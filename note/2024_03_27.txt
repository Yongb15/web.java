지금 당장은 배열의 도우미 기능을 지닌 것으로 생각하자

ex) Scanner sc = new Scanner(System.in); - 90%

- sc.next();

ex) String s = " " - 10%

객체지향 프로그래밍
- 객체를 지향하는 프로그래밍 방법

객체란 - 우리 실생활에 존재하는 모든 것을 의미한다.

일반적으로 상태를 표현할 수 있고, 행동으로 실행할 수 있는 모든 것
객체를 중심으로 프로그램 구조를 설계하고 프로그래밍 하는 것

클래스란 - 객체를 정의하기 위한 설명서
           ㄴ 클래스를 기반으로 객체를 생성
           ㄴ 하나의 클래스로 여러 개의 객체를 생성할 수 있다.

클래스의 종류
- 실행용 클래스
    ㄴ 프로그램 전체에서 단 하나의 클래스로 프로그램의 실행을 맡고있다.
    ㄴ main 메서드를 갖고 있으며, 다른 클래스에서 참조되지 않는다.

- 객체 생성용 클래스
    ㄴ 다른 클래스에서 사용할 목적으로 선언되는 클래스

하나의 클래스가 위 두 가지 용도의 역할을 모두 수행할 수도 있다.
하지만 유지 보수와 객체 지향 프로그래밍의 특징인 모듈화를 고려해 별도로 분리하여 작성하는 것을 권장한다.
일반적으로 하나의 프로그램에서 실행용 클래스 1개를 제외한 나머지 클래스는 모두 참조용 클래스이다.

클래스의 선언

접근제한자 class 클래스명{
	
}

클래스 이름을 작성하는 규칙
- 영어 대소문자를 사용할 수 있으며, 첫 글자는 대문자를 사용한다.
- 숫자를 사용할 수 있으나 첫 글자로는 사용할 수 없다.
- 특수문자는 $, _만 사용가능하다
- 자바 예약어(if, for, while, weitch 등등)는 사용할 수 없습니다.
